# Frontend Design Skill

Dieser Skill leitet die Erstellung von hochwertigen, produktionsreifen Frontend-Interfaces an, die generische "AI-Slop" Ästhetik vermeiden. Implementiere echten, funktionierenden Code mit außergewöhnlicher Aufmerksamkeit für ästhetische Details und kreative Entscheidungen.

## Design Thinking

Vor dem Coden den Kontext verstehen und eine **MUTIGE** ästhetische Richtung wählen:

- **Zweck**: Welches Problem löst dieses Interface? Wer nutzt es?
- **Ton**: Wähle einen Extrempunkt: brutal minimal, maximalistisch, retro-futuristisch, organisch/natürlich, luxuriös/raffiniert, verspielt/toy-like, editorial/magazine, brutalistisch/raw, art deco/geometrisch, soft/pastell, industriell/utilitaristisch etc.
- **Constraints**: Technische Anforderungen (Framework, Performance, Accessibility)
- **Differenzierung**: Was macht es UNVERGESSLICH? Was ist das eine Detail dass jemand in Erinnerung behalten wird?

**KRITISCH**: Wähle eine klare konzeptuelle Richtung und führe sie mit Präzision aus. Mutiger Maximalismus und raffinierter Minimalismus funktionieren beide – der Schlüssel ist Intentionalität, nicht Intensität.

## Frontend Ästhetik Guidelines

### Typografie

- Wähle Fonts die schön, einzigartig und interessant sind
- **VERMEIDE** generische Fonts wie Arial und Inter
- Markante Display-Font mit raffinierter Body-Font paaren
- Unerwartete, charaktervolle Font-Choices die die Ästhetik heben

### Farbe & Theme

- Sich auf eine kohärente Ästhetik committen
- CSS Variables für Konsistenz nutzen
- Dominante Farben mit scharfen Akzenten schlagen schüchterne, gleichmäßig verteilte Paletten
- **NIEMALS** Standard-Purple-Gradients auf weißem Hintergrund

### Motion & Animation

- Animationen für Effekte und Micro-Interactions nutzen
- CSS-only Lösungen für HTML bevorzugen
- Fokus auf High-Impact Momente: ein orchestrierter Page-Load mit gestaffelten Reveals
- Scroll-triggering und Hover-States die überraschen
- `animation-delay` für gestaffelte Animationen

### Spatial Composition

- Unerwartete Layouts, Asymmetrie, Überlappung
- Diagonaler Flow, Grid-brechende Elemente
- Großzügiger Negativraum ODER kontrollierte Dichte

### Hintergründe & visuelle Details

- Atmosphäre und Tiefe statt einfache Solid-Colors
- Gradient Meshes, Noise-Texturen, geometrische Patterns
- Layered Transparencies, dramatische Schatten
- Dekorative Borders, Custom Cursors, Grain-Overlays

## ❌ NIEMALS verwenden

- Overused Font Families (Inter, Roboto, Arial, System Fonts)
- Klischee Farbschemen (insbesondere Purple Gradients auf Weiß)
- Vorhersagbare Layouts und Component-Patterns
- Cookie-Cutter Designs ohne kontextspezifischen Charakter
- Gleiche Choices in verschiedenen Generationen (z.B. Space Grotesk)

## ✅ Immer beachten

- Kreativ interpretieren und unerwartete Choices machen
- Zwischen Light und Dark Themes variieren
- Verschiedene Fonts, verschiedene Ästhetiken
- Implementation-Komplexität an ästhetische Vision anpassen
- Maximalist = elaborate Code, viele Animationen
- Minimalist = Restraint, Präzision, subtile Details
